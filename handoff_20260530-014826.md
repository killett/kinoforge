## 1. What kinoforge is

A vendor-agnostic video-generation provisioning and orchestration system, driven entirely by
YAML config. The mission (from `SPEC.md`) is to make compute providers, model sources, and
generation engines all swappable plugins behind a registry — never hard-coded — so future
provider/engine/model swaps are config edits, not rewrites.

This release ships the **provisioning + single-clip generation layer plus every seam** the
future long-form pipeline (prompt splitter, continuity/stitching, audio sync, concurrent
scheduler, keyframe stage) will bolt onto. The deferred layers are explicitly NOT implemented;
their interfaces and one real path each are.

The three swappable axes:

| Axis | Built |
|---|---|
| Compute provider | `LocalProvider`, `RunPodProvider` (pod + serverless), `SkyPilotProvider` |
| Model source | `HTTPSource`, `CivitAISource`, `HuggingFaceSource` |
| Generation engine | `FakeEngine`, `ComfyUIEngine`, `DiffusersEngine`, `HostedAPIEngine` |

Plus a `LocalArtifactStore` for the artifact-store axis, leaving S3/GCS as a deferred plugin.

## 3. Architectural decisions (locked)

### 3.1 Core never imports a concrete adapter (Principle 1)
- `src/kinoforge/core/*` depends only on `core/interfaces.py`, `core/errors.py`, and `stores/base.py`.
- Adapters self-register at import time via `register_provider` / `register_engine` /
  `register_source` / `register_store`.
- The CLI imports concrete adapter packages **in exactly one place** (`src/kinoforge/_adapters.py`)
  to trigger self-registration. Nothing else does.
- Enforced by `tests/test_core_invariant.py` (subprocess sys-modules check + static text scan).

### 3.2 Eight open questions, resolved in DESIGN.md §1

| Topic | Decision |
|---|---|
| Submit/result vs Pool/Future | `submit→job_id` + `result(job_id)` is the backend contract; `BackendPool` owns Futures. `SequentialPool` returns an already-resolved Future. |
| `models:` per engine | Always parsed for `CapabilityKey` identity; downloaded only when `engine.requires_local_weights` is True. Hosted: validation-only. |
| `params` vs `spec` | `params` = engine-neutral knobs every engine honours identically (fps/resolution/steps/seed/cfg/frames). `spec` = engine-only payload. `validate_spec` guards `spec` before dispatch. |
| Profile cache location | JSON via `ArtifactStore`, default `.kinoforge/profiles/`, path overridable. Single-flight via in-process `threading.Lock` + per-key `Event`; cross-process coordination deferred. |
| Serverless caps | `max_workers`, per-request deadline, `max_in_flight`. `budget` is the reactive backstop. |
| Artifact GC | Run-scoped `run_id` namespaces + explicit `kinoforge gc [--run ID] [--older-than DUR]`. No daemon. |
| Conditioning roles | `init_image` / `first_frame` / `last_frame` / `drive_audio` / `source_video`. `MODE_ROLE_REQUIREMENTS` maps mode → required roles in ONE place. |
| Under-use warning | Declared-only flags. No inference from probed modes. Undeclared flags default False + a WARNING naming the `CapabilityKey`. |

### 3.3 Discovery ordering (guaranteed, explicit in code)
1. `resolve(key)` cache read.
2. On miss → `discover(key, engine, backend)` ONCE under single-flight lock → persist.
3. `validate_request(profile, request)` — mode + role-authoritative.
4. Splitter stub → 1 segment (DEFERRED).
5. Provision backend.
6. `verify(profile, backend)` re-probes probeable fields.
7. On `CapabilityMismatch`: **fail-hard** — re-raise + `provider.destroy_instance(id)` exactly once.

### 3.4 Cost-safety
- **Invariant** (universal, non-negotiable): bounded instance lifetime, idle reap, hung-job abort, confirmed teardown, no orphans.
- **Mechanism** is provider-specific:
  - `RunPodProvider`: in-pod self-terminator installed at create; least-privilege terminate-only credential (`RUNPOD_TERMINATE_KEY`, never `RUNPOD_API_KEY`); serverless mode gets proactive caps instead.
  - `SkyPilotProvider`: maps `idle_timeout` → SkyPilot autostop. Custom timer model not replicated (documented trade-off).
  - `LocalProvider`: injectable clock for fully offline tests.
- Three timers, each one concern: `idle_timeout` (warm reuse), `job_timeout` (per-clip), `max_lifetime` (graceful drain). `effective_deadline = n*job_timeout + time_buffer`.
- Universal: instance tags + persistent `Ledger`, `destroy_confirmed` (poll + retry + alert), `BudgetTracker.enforce` tears down BEFORE raising.

### 3.5 Capability key
`CapabilityKey(base_model, loras=(), engine="", precision="")`.
- `derive()` = sha256 over `json.dumps([...], ensure_ascii=False)`. JSON escapes guarantee distinct tuples never collide (a code-review finding caught the original `\x1f`/`\x1e` separator scheme and forced this fix — see commit `7e70a57`).
- VAE entries are excluded entirely (decode-only, doesn't change generation capability).
- LoRA order matters and contributes to the hash.
- Exactly ONE `kind: base` per config enforced at load time (caught by a code-review finding — see commit `94afa3e`).

### 3.6 Dependencies, stdlib-first
- Runtime: `pydantic v2`, `PyYAML`, optional `skypilot` (lazy-imported only inside `providers/skypilot/`).
- HTTP: `urllib` everywhere (CivitAI, HF, downloader, RunPod, ComfyUI graph API, Diffusers server, Hosted API). No `requests`, no `huggingface_hub`.
- Logging: stdlib `logging` via a thin `core/logging.get_logger()`.
- Dev: `ruff`, `mypy`, `pytest`, `pytest-cov`, `pre-commit`, `types-pyyaml`. All via pixi.

## 4. Patterns established (use these for future work)

### 4.1 Injected I/O seams
Every adapter takes its HTTP / subprocess / filesystem dependencies as constructor parameters with stdlib-real defaults. Tests pass spies; production uses the real defaults. Examples:
```python
ComfyUIEngine(run_cmd=_subprocess_run, http_post=_urllib_post_json,
              http_get=_urllib_get_json, file_exists=Path.exists,
              route_file=shutil.move, sleep=time.sleep, ...)
RunPodProvider(creds=None, http_post=_urllib_post_json,
               http_get=_urllib_get_json, sleep=time.sleep, ...)
CivitAISource(fetch=_urllib_fetch_json)
HTTPSource()  # downloader handles transport, source is dumb
```
**Rule:** NO test makes real network/subprocess/git/GPU calls. The `http_server` fixture in `tests/conftest.py` is the one allowed loopback exception (Range-aware `ThreadingHTTPServer` on `127.0.0.1:0`), used by the downloader tests.

### 4.2 Self-registration on import
Every concrete adapter ends its module with:
```python
registry.register_engine("comfyui", lambda: ComfyUIEngine())
registry.register_provider("runpod", lambda: RunPodProvider())
registry.register_source(HTTPSource())             # instance, not factory
registry.register_store("local", lambda: LocalArtifactStore(Path(".kinoforge")))
```
Sources register as instances because dispatch is by `handles(ref)`. Providers, engines, stores register as **zero-arg factories** so re-imports are idempotent and construction is lazy.

### 4.3 Source dispatch by behaviour, not key
`source_for_ref(ref)` asks each registered source `handles(ref)` and returns the first match. A source may claim multiple schemes (`http`/`https`). Re-registering a source with the same `scheme` replaces the prior entry. This was the load-bearing distinction caught during Task 2 (PROGRESS.md briefly sketched a `register_source(scheme, cls)` API — the plan's `register_source(instance)` + `handles()` design won out, see commit `f33ec13`).

### 4.4 Stage protocol + pool-swap
`pipeline.Stage` is a `@runtime_checkable` Protocol re-exported from `core.interfaces`. The single concrete stage today (`GenerateClipStage`) talks only to `BackendPool` + `ArtifactStore` + `ModelProfile`. Swapping `SequentialPool` for a future concurrent pool is a constructor-arg change, not a refactor (proven by the `_ListPool` test).

### 4.5 Strategy as a pure function
`core.strategy.decide(profile, segments, params, spec) -> list[GenerationJob]`. Native-extension True → one job carrying all N segments; False → N single-segment jobs (continuity/stitching DEFERRED). Segment params override base params (segment-wins merge). `spec["_audio_mode"]` marker carries the joint vs separate audio decision; separate-audio path is DEFERRED but the seam is wired.

### 4.6 Validation as a pure function
`core.validation.validate_request(profile, request, *, accepted_kinds)` enforces mode ∈ `supported_modes` and the `MODE_ROLE_REQUIREMENTS[mode]` contract. Returns a NEW `GenerationRequest` (via `dataclasses.replace`) — never mutates input. Single-asset modes default a lone bare image to `init_image`; multi-asset modes require explicit roles.

### 4.7 TDD red-first, every task
Each task wrote the failing test FIRST, ran it, confirmed FAIL, then implemented. The `test-design` skill conventions held: bug-catch comments on non-trivial assertions, no implementation mirroring, no over-mocking, no weak assertions.

### 4.8 Durability
- `git` is the source of truth, never the conversation.
- `PROGRESS.md` updated and committed after every task — the single recovery index.
- The `superpowers-extended-cc:subagent-driven-development` skill drove execution: one fresh implementer subagent per task, spec compliance review, code quality review, fix loops until clean.

### 4.9 Code-review findings worth knowing
Two foundational invariants were patched in response to code-review:
- `CapabilityKey.derive()` switched from `\x1f`-separated strings to `json.dumps(...)` (commit `7e70a57`).
- `Config.capability_key()` requires **exactly one** `kind: base` entry; zero or many → `ConfigError` at load time (commit `94afa3e`).

Plus one over-scope rollback:
- A blanket `ignore_errors = true` on `tests.*` mypy override was reverted in favour of targeted `# type: ignore[arg-type, return-value, comparison-overlap]` on the deliberate duck-typed test fakes (commit `c26c8ea`).

### 4.10 `.gitignore` lesson
Scaffolded `.gitignore` only excluded `.claude-project-id`. Executor subagents that `git add src/kinoforge/...` after pixi populated `__pycache__/` ended up tracking 30 `.pyc` files. The merge commit was preceded by a `.gitignore` expansion + untrack (commit `aa66a8f`). Future projects: start with a full standard Python `.gitignore`.

## 6. What works today (Definition of done coverage)

Every SPEC.md "Definition of done" bullet has at least one passing test. Highlights:
- `kinoforge deploy --config local-fake.yaml --dry-run` prints a vendor/engine-neutral plan with zero network calls.
- End-to-end against `LocalProvider + FakeEngine + LocalArtifactStore`: `generate` produces a clip Artifact whose `uri` is in the store, bytes round-trip via `store.get_bytes`.
- Swapping `compute.provider` (runpod ↔ local) and `engine.kind` (comfyui ↔ diffusers) is config-only.
- `HostedAPIEngine` runs the generate path with `instance=None`.
- Capability discovery: changing models changes `max_segment_seconds` with no code change.
- Plan-time cache hit reads the profile with no backend touched.
- Capability key distinctness: base vs base+lora vs same base under two engines → distinct cache entries.
- Single-flight discovery: two concurrent threads → exactly ONE `inspect_capabilities` call.
- Fail-hard on drift: `verify` mismatch raises `CapabilityMismatch` AND invokes `destroy_instance` exactly once.
- `find_offers` filtering (vram, cuda semantic compare, pod-only cost filter, gpu_preference ordering).
- Mode + role-authoritative request validation incl. flf2v dual-image + i2v lone-image default.
- Strategy both branches: native → 1 job/N segs; non-native → N single-seg jobs.
- Cost-safety items a–i (warm reuse, drain, hung-job detection scaling with segments, in-flight liveness vs idle dead-man, confirmed teardown, sweeper, budget kill, config rejects nonsensical timers, defaults applied).
- CLI's duplicate-pod refusal.

## 8. Known minor limitations (not blockers)

1. `JsonProfileCache._reconstruct_uri` peeks at `LocalArtifactStore._path` via `hasattr` to rebuild URIs on cross-restart reads. The clean fix is to add `ArtifactStore.uri_for(run_id, name) -> str` to the ABC. Required when shipping S3/GCS stores.
2. `cli._cmd_status` queries only the in-process provider state, not the ledger. Fine for the local-fake path; will need ledger-driven status for distributed providers.
3. `provisioner.provision` is invoked with `cfg: Config` from the CLI but typed `_ProvisionConfig` (a Protocol); a `# type: ignore[arg-type]` rides at the call site for mypy's generic variance. Acceptable; could be cleaned with a more precise Protocol or a `TypeVar` if it bites later.
4. `RunPodProvider.find_offers` is structurally correct but its real RunPod GPU-types REST shape is a stub — real-cloud usage will need a small JSON adapter once you point it at a live key.
5. SkyPilot provider ships the structure + injected client seam. The lazy `_get_sky()` real path is wired but unexercised (tests cover the seam, not the real `sky` SDK). First real-cloud use will surface any SDK-version drift.
